# コントリビューションガイド

本リポジトリへの貢献方法について説明します。

## 🤝 貢献方法

### コード・ドキュメントの追加

1. **イシューの作成**
   - 新しい機能や改善提案がある場合は、まずイシューを作成
   - 既存のイシューを確認し、重複を避ける

2. **ブランチの作成**
   ```bash
   git checkout -b feature/new-feature
   ```

3. **変更の実施**
   - コードやドキュメントを追加・修正
   - コミットメッセージは明確に記述

4. **プルリクエスト**
   - 変更をプッシュし、プルリクエストを作成
   - 変更内容を詳しく説明

## 📝 コーディング規約

### Python

- PEP 8に従う
- docstringを記述
- 型ヒントを使用（Python 3.6+）

```python
def calculate_score(data: List[float], threshold: float = 0.5) -> float:
    """
    スコアを計算する関数
    
    Args:
        data: 入力データのリスト
        threshold: 閾値（デフォルト: 0.5）
    
    Returns:
        計算されたスコア
    """
    pass
```

### ドキュメント

- Markdown形式で記述
- 見出しは階層的に使用
- コードブロックには言語を指定

## 🧪 テスト

新しいコードを追加する場合は、テストも追加してください：

```bash
# テストの実行
pytest tests/

# カバレッジの確認
pytest --cov=src tests/
```

## 📋 コミットメッセージ

コミットメッセージは以下の形式に従ってください：

```
[タイプ] 簡潔な説明

詳細な説明（必要に応じて）

関連イシュー: #123
```

**タイプの例:**
- `feat`: 新機能
- `fix`: バグ修正
- `docs`: ドキュメントのみの変更
- `style`: コードの動作に影響しないスタイル変更
- `refactor`: リファクタリング
- `test`: テストの追加・修正
- `chore`: ビルドプロセスやツールの変更

## 🔍 コードレビュー

プルリクエストは以下の点をチェックされます：

- [ ] コードが正しく動作する
- [ ] テストが含まれている
- [ ] ドキュメントが更新されている
- [ ] コーディング規約に従っている
- [ ] 不要なファイルが含まれていない

## 🚫 やってはいけないこと

- 機密情報（パスワード、APIキーなど）をコミットしない
- 大容量のバイナリファイルを直接コミットしない（Git LFSを使用）
- 既存のコードを不必要に変更しない
- コミットメッセージを曖昧にしない

## 📞 質問・サポート

質問がある場合は：

1. まず既存のイシューを検索
2. 該当する内容がなければ、新しいイシューを作成
3. 緊急の場合は、メンテナに直接連絡
